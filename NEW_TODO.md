# New TODO

1. Constrain theorem-stating reviewer decisions after `INVALID`.
On an `INVALID` theorem-stating attempt, the reviewer currently still runs and can emit the full theorem-stating decision set. That is too permissive. In particular, `ADVANCE_PHASE` should not be legal after an invalid worker attempt, and any other decision that assumes a valid package should be either forbidden or explicitly downgraded by the supervisor. The valid control surface on an invalid attempt should be narrow: review the dirty worktree, give corrective guidance, optionally request `NEED_INPUT`, and optionally reset to one of the supervisor-approved valid checkpoints.

2. Make theorem-stating worker handoff authoritative about new nodes.
Right now the worker handoff requires `new_nodes`, but the supervisor still discovers created nodes from the filesystem and can register nodes that were not listed in the handoff. That weakens the contract between the worker artifact and the supervisor. We should decide and enforce a cleaner model: either unlisted node creations are deterministically invalid, or the handoff schema should be expanded to reflect the real authoritative set of created/modified/deleted tablet nodes. The important invariant is that the supervisor should not silently accept structural changes that the worker artifact did not actually declare.

3. Give invalid theorem-stating attempts stronger historical representation.
At the moment an invalid theorem-stating attempt can preserve a dirty worktree and get reviewed, but it still does not receive the same exact historical treatment as a valid cycle in the main repo. Chats may be checkpointed separately, but reconstructing the exact invalid tablet/worktree state later is weaker than reconstructing a valid worker or verification checkpoint. We should decide whether invalid attempts should get explicit exact checkpoints in the main repo, or some other equally reliable historical representation, so that invalid theorem-stating work is inspectable and replayable without pretending it was a valid cycle.
